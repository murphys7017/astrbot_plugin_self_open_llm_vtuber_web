# 阶段更新总结（截至 2026-03-23）

本文汇总近期围绕桌宠模式做的结构性改造与问题修复，重点覆盖交互窗拆分、输入/麦克风稳定性、以及 Live2D 兼容性补丁。

## 1. 桌宠窗口架构调整

- 将原本“桌宠全屏透明窗口 + 内嵌交互区”的方案改为双窗口：
- 主窗口：承载 Live2D 模型展示与拖拽。
- 交互窗口（Pet Overlay）：独立承载输入框、状态文本、麦克风/打断按钮。
- Pet 模式切换时自动创建/销毁交互窗口，并保持顶层显示。

涉及文件：
- `src/main/window-manager.ts`
- `src/renderer/src/App.tsx`
- `src/renderer/src/components/electron/pet-overlay-window.tsx`

## 2. 主窗与交互窗通信桥接（IPC）

- 新增 Pet Overlay 通道，支持：
- 交互窗 -> 主窗：发送文本、切换麦克风、打断。
- 主窗 -> 交互窗：AI 状态、最近回复、麦克风状态同步。
- 通过单连接转发，避免“第二个窗口再开一条 WebSocket”造成状态分裂。

涉及文件：
- `src/preload/index.ts`
- `src/preload/index.d.ts`
- `src/renderer/src/env.d.ts`
- `src/renderer/src/hooks/utils/use-pet-overlay-bridge.ts`

## 3. 交互窗可独立拖动与布局优化

- 交互窗改为可独立拖动，不再强绑定跟随主窗移动。
- 新增顶部拖拽条（`WebkitAppRegion: drag`），交互控件区域保持 `no-drag`。
- 初始位置改为主窗侧边优先，避免默认遮挡模型。
- 支持按内容高度动态调整交互窗高度（最小/最大高度限制）。
- 修复交互窗滚动条异常（根节点 overflow、`100vw` 溢出问题）。

涉及文件：
- `src/main/window-manager.ts`
- `src/renderer/src/components/electron/pet-overlay-window.tsx`

## 4. 输入与中文输入法（IME）稳定性修复

- 修复 Pet 模式下输入焦点不稳定导致中文输入异常的问题：
- 输入框 `focus/blur` 显式通知主进程，稳定窗口可聚焦/可交互状态。
- 补充 `nativeEvent.isComposing` 判断，避免中文候选期间误触发 Enter 发送。

涉及文件：
- `src/main/window-manager.ts`
- `src/preload/index.ts`
- `src/preload/index.d.ts`
- `src/renderer/src/components/electron/input-subtitle.tsx`
- `src/renderer/src/hooks/footer/use-text-input.tsx`

## 5. 麦克风相关修复

- 修复 Pet 模式下首次点击麦克风不生效问题：
- 交互窗先尝试权限预请求，再发起主窗麦克风切换。
- 修复窗口模式开麦回归：窗口模式恢复为直接 `startMic()`。
- 修复 ASR 设置页 `Microphone request timed out in UI` 的未捕获 Promise：
- UI 超时计时器在任务完成后会清理。
- 超时/失败统一落日志，不再抛出未处理异常。

涉及文件：
- `src/renderer/src/components/electron/pet-overlay-window.tsx`
- `src/renderer/src/hooks/utils/use-pet-overlay-bridge.ts`
- `src/renderer/src/hooks/utils/use-mic-toggle.ts`
- `src/renderer/src/components/sidebar/setting/asr.tsx`

## 6. 鼠标穿透与事件监听稳定性

- 桌宠鼠标穿透逻辑重构：减少高频 hit-test 与焦点抢占抖动。
- 将 `removeAllListeners` 改为精确 `removeListener`，避免误删其他监听器。

涉及文件：
